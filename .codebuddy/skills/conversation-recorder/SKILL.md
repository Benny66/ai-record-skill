# Conversation Recorder — AI 协作开发经验引擎

## Purpose

三层知识架构，将碎片化的 AI 对话转化为可积累、可检索、可复用的开发经验：

```
第一层：对话原始记录（每轮自动）
     ↓ 生成日报时回顾式提炼
第二层：日报 + 经验卡片（每日沉淀）
     ↓ 生成周报/月报时蒸馏
第三层：经验手册（长期精华，去重合并）
```

## When to Use

本 Skill 加载后**持续生效**，覆盖以下场景：

1. **自动对话记录**：每轮对话结束后自动触发（含正常完成、用户终止、异常中断）
2. **日报生成**：用户说触发词时，回顾当日对话 + 提炼经验卡片
3. **周报/月报生成**：跨天汇总 + 从日报经验卡片中蒸馏精华到经验手册
4. **经验手册维护**：更新/查看/搜索长期积累的经验
5. **经验推荐**：对话过程中如命中已有经验，在回复末尾附一句提示

## 核心规则（必须严格遵守）

### 规则 1：每轮对话结束后必须执行记录

在完成对用户请求的回复后（**无论回复是否完整**），**必须立即执行以下记录流程**，不可跳过、不可遗忘：

1. 确定当前日期，格式 `YYYY-MM-DD`
2. 确定日志文件路径：`{project_root}/conversation-logs/{YYYY-MM-DD}.md`
3. 若 `conversation-logs/` 目录不存在，先创建
4. 若日志文件不存在，创建并写入标题 `# 对话记录 - {YYYY-MM-DD}`
5. 若日志文件已存在，读取文件确定当前最大对话编号
6. 判断本轮对话的完成状态（见规则 6）
7. 构造记录内容（格式见规则 2，不完整对话格式见规则 6）
8. 将记录内容追加到日志文件末尾

**关键原则**：记录的触发条件是"AI 获得了执行机会"，而非"AI 完整回答了问题"。只要 AI 在本轮有机会执行工具调用（包括写文件），就必须执行记录。

### 规则 2：记录内容格式（严格遵守）

每轮对话的记录格式如下：

```markdown
---

## [{HH:MM}] 对话 #{N}

### 场景分类
`{分类标签}`

### 用户输入
> {用户的原始自然语言输入，原封不动，保留换行}
> {如果用户输入有多行，每行前面都加 > }

### AI 反馈摘要
- {要点1}：{具体内容}
- {要点2}：{具体内容}
- {要点3}：{具体内容}
```

**字段说明**：
- `{HH:MM}`：当前时间，24小时制，精确到分钟
- `{N}`：当日对话序号，从 1 开始递增
- `{分类标签}`：核心使用场景分类，见规则 3，使用反引号包裹
- 用户输入：使用 Markdown blockquote（`>`）格式，**完全复制用户原始文本，不做任何修改、不省略、不概括**
- AI 反馈摘要：提取 **2-5 个关键要点**，每个要点一行

### 规则 3：场景分类与 AI 反馈摘要

#### 3.1 核心使用场景分类

每轮对话**必须**归入以下 5 个分类之一：

| 编号 | 分类标签 | 判定标准（满足任一即归入） |
|------|---------|------------------------|
| 1 | `快速代码生成与复用` | 生成新代码、创建新文件/模块、脚手架搭建、模板复用、批量生成 |
| 2 | `代码纠错与实时调试排障` | bug/报错修复、调试排障、运行失败分析、异常排查、错误信息解读 |
| 3 | `代码优化与规范整改` | 性能优化、重构、规范检查整改、安全漏洞修复、架构调整 |
| 4 | `学习赋能与技术落地` | 概念解释、原理讲解、方案选型、最佳实践、技术调研、知识问答 |
| 5 | `辅助文档与注释产出` | 生成/修改文档、编写注释、README、API 文档、报告生成 |

**分类判定原则**：
1. 以用户主要意图为准
2. 混合场景取耗时最多/最核心的分类
3. 无法判断时默认 `学习赋能与技术落地`
4. 技能/工具加载类归入 `辅助文档与注释产出`

#### 3.2 AI 反馈摘要的提取规则

按优先级提取 2-5 个要点：
1. **文件操作**：创建、修改、删除了哪些文件
2. **关键决策**：技术或业务决策
3. **执行命令**：运行了什么命令及结果
4. **给出建议**：建议或注意事项
5. **回答问题**：核心结论
6. **流程推进**：推进了哪个阶段

格式：`- {分类}：{一句话描述}`

### 规则 4：日报生成（含经验卡片提炼）

**触发词**：`生成日报`、`今日总结`、`daily report`、`对话日报`、`生成今日日报`

**日报生成步骤**：

1. 确定当前日期 `YYYY-MM-DD`
2. 调用脚本提取结构化数据：`python {SKILL_DIR}/scripts/report.py {PROJECT_ROOT} --type daily --date {YYYY-MM-DD}`
3. 读取 `conversation-logs/{YYYY-MM-DD}.md` 文件
4. 若文件不存在或为空，提示"今日暂无对话记录"，不创建空日报
5. 基于脚本输出的 JSON 数据 + 原始对话内容，回顾全天对话提炼经验卡片
6. 若 `conversation-logs/reports/` 目录不存在，先创建
7. 生成日报内容（含经验卡片章节），填充脚本数据到模板
8. 写入 `conversation-logs/reports/{YYYY-MM-DD}-report.md`
9. 同时向用户展示日报内容
10. **注意**：日报中的经验卡片不自动写入经验手册，经验手册的更新在"更新经验手册"或"生成周报/月报"时统一蒸馏

**日报格式**：

```markdown
# 日报 - {YYYY-MM-DD}

## 概览
- **日期**：{YYYY-MM-DD}
- **对话总数**：{N} 轮
- **涉及项目**：{项目名称}
- **新增经验**：{M} 条

## 场景分布

| 场景分类 | 对话数 | 占比 |
|---------|--------|------|
| 快速代码生成与复用 | {N1} | {P1}% |
| 代码纠错与实时调试排障 | {N2} | {P2}% |
| 代码优化与规范整改 | {N3} | {P3}% |
| 学习赋能与技术落地 | {N4} | {P4}% |
| 辅助文档与注释产出 | {N5} | {P5}% |

## 对话摘要

| # | 时间 | 场景分类 | 用户问题概括 | AI 执行摘要 |
|---|------|---------|-------------|------------|
| 1 | {HH:MM} | {分类} | {≤20字} | {≤30字} |

## 关键产出
- **创建文件**：{列出当日新建的文件}
- **修改文件**：{列出当日修改的文件}
- **关键决策**：{列出当日重要决策}

## 当日经验卡片

{以下为 AI 回顾全天对话后提炼的经验，非每轮都有，仅提取真正值得沉淀的}

### EXP-{YYYYMMDD}-{NN}：{一句话标题}
- **类型**：`{经验类型}`
- **场景分类**：{5种分类之一}
- **经验**：{2-5句话描述具体经验}
- **反模式**：{对应的错误做法，如适用}
- **可复用度**：{★ 到 ★★★★★}
```

**经验卡片提炼规则**：

经验卡片在生成日报时**回顾式提炼**，不是每轮对话实时提取。AI 回顾一整天的对话后，仅提取真正值得沉淀的经验。一天可能 0 条也可能 5 条。

**经验类型（4 大类）**：

| 大类 | 类型标签 | 提取信号 |
|------|---------|---------|
| **提示词与对话策略** | `提示词技巧` | 发现有效的提问模式、指令格式 |
| | `Token优化` | 发现省 token 的方法 |
| | `上下文管理` | 长对话管理、会话切换策略 |
| | `迭代策略` | 与 AI 迭代的有效方式 |
| **代码开发经验** | `踩坑记录` | bug 修复、走弯路、调试教训 |
| | `技术决策` | 技术选型、架构决策 |
| | `开发最佳实践` | 验证有效的编码做法 |
| **AI 风险认知** | `幻觉识别` | AI 编造不存在的内容 |
| | `认知偏差` | AI 给出有偏见/片面的回答 |
| | `安全防范` | AI 输出需要人工验证的风险点 |
| **工具与流程** | `工具技巧` | CodeBuddy/Skill/插件的高效用法 |
| | `Skill开发经验` | Skill 编写、调试、优化的经验 |
| | `流程优化` | 更优的工作流程 |

**提炼原则**：
1. **宁缺毋滥**：只提取有实际复用价值的经验，日常确认/简单操作不提取
2. **回顾视角**：站在"如果明天遇到类似场景"的角度判断是否值得记录
3. **反模式必记**：如果当天犯了错（如 AI 理解偏差导致返工），一定要记录反模式
4. **合并同类**：同一天内多轮对话指向同一个经验点的，合并为一条

### 规则 5：周报 / 月报生成 + 经验蒸馏

**周报触发词**：`生成周报`、`weekly report`、`本周总结`
**月报触发词**：`生成月报`、`monthly report`、`本月总结`

**周报生成步骤**：

1. 确定本周日期范围（周一到当日）
2. 调用脚本提取周度数据：`python {SKILL_DIR}/scripts/report.py {PROJECT_ROOT} --type weekly --date {YYYY-MM-DD}`
3. 调用统计分析脚本：`python {SKILL_DIR}/scripts/stats.py {PROJECT_ROOT} --from {周一日期} --to {当日日期}`
4. 读取该范围内所有日报文件 `reports/{YYYY-MM-DD}-report.md`，提取经验卡片
5. 若无任何日报，提示无数据
6. 基于脚本数据 + 日报经验卡片，汇总生成周报
7. 写入 `reports/{YYYY}-W{WW}-weekly.md`
8. 执行经验蒸馏（见下方经验蒸馏规则）

**周报格式**：

```markdown
# 周报 - {YYYY}年第{WW}周（{MM-DD} ~ {MM-DD}）

## 概览
- **对话总数**：{N} 轮（日均 {avg} 轮）
- **活跃天数**：{D} 天
- **新增经验**：{M} 条
- **经验手册总条目**：{T} 条

## 场景分布趋势

| 场景分类 | 本周对话数 | 本周占比 | 趋势 |
|---------|----------|---------|------|
| 快速代码生成与复用 | {N1} | {P1}% | {↑/↓/→} |
| 代码纠错与实时调试排障 | {N2} | {P2}% | {↑/↓/→} |
| 代码优化与规范整改 | {N3} | {P3}% | {↑/↓/→} |
| 学习赋能与技术落地 | {N4} | {P4}% | {↑/↓/→} |
| 辅助文档与注释产出 | {N5} | {P5}% | {↑/↓/→} |

## 每日摘要

| 日期 | 对话数 | 主要工作 | 新增经验 |
|------|--------|---------|---------|
| {MM-DD} | {N} | {一句话概括} | {M}条 |

## 本周关键产出
- {汇总本周创建/修改的关键文件和决策}

## 本周经验精选

{从本周所有日报的经验卡片中，挑选最有价值的 3-5 条}

### {EXP-ID}：{标题} [★★★★★]
{经验内容}

## AI 使用效率分析
- **一次成功率**：{首轮对话就达到目标的比例}
- **平均迭代轮数**：{达到目标平均需要几轮对话}
- **终止/中断率**：{不完整对话占比}
- **改进建议**：{基于数据给出的 AI 使用改进建议}
```

**月报**格式与周报类似，额外增加：
- 月度经验增长曲线（文字描述每周新增条数）
- 月度场景分布变化趋势
- 月度 Top 10 经验精选
- 下月 AI 使用改进计划

**经验蒸馏规则**：
生成周报/月报时，AI 自动将期间内的经验卡片蒸馏到经验手册：
1. 读取经验手册 `experience-handbook.md`
2. 对比新经验与已有条目
3. 全新经验 → 追加到手册对应分类
4. 与已有条目相似 → 合并，标注"多次验证"，提升可复用度星级
5. 与已有条目矛盾 → 更新旧条目，标注修正原因

### 规则 6：不完整对话的记录（终止 / 中断 / 重新提问）

对话过程中可能出现以下非正常结束的情况，**都必须记录**：

#### 场景 A：用户手动终止 AI 回答

**识别方式**：上下文中存在被截断的 AI 回复，紧接着是用户新消息。

```markdown
---

## [{HH:MM}] 对话 #{N} ⚠️ 用户终止

### 场景分类
`{分类标签}`

### 用户输入
> {原始输入，原封不动}

### AI 反馈摘要（回答被用户终止）
- 状态：用户手动终止了 AI 回答
- 已完成部分：{概括已输出内容}
- 未完成部分：{概括未完成的事}
```

#### 场景 B：AI 回答因异常中断

```markdown
---

## [{HH:MM}] 对话 #{N} ⚠️ 异常中断

### 场景分类
`{分类标签}`

### 用户输入
> {原始输入，原封不动}

### AI 反馈摘要（回答异常中断）
- 状态：AI 回答因异常中断（{中断原因}）
- 已完成部分：{概括已输出内容}
```

#### 场景 C：终止后重新提问

先按场景 A 记录被终止的对话，再正常记录新对话（编号递增）。

#### 通用规则

1. 不完整对话也占用对话编号，后续递增
2. AI 完全没来得及输出时，`已完成部分` 写"AI 尚未输出任何内容"
3. 时间戳使用原始时间，不是补记时间
4. 补记时机：用户发送下一条消息时，先补记再记新对话
5. 日报中标注"（被用户终止）"或"（异常中断）"

### 规则 7：经验手册维护

**经验手册路径**：`{project_root}/conversation-logs/experience-handbook.md`

**触发词**：
- 更新经验手册：`更新经验手册`、`整理经验`、`沉淀经验`
- 查看经验手册：`查看经验`、`经验手册`、`有什么经验`
- 搜索经验：`搜索经验 {关键词}`

**更新经验手册**时 AI 执行：
1. 读取经验手册
2. 读取最近未合入手册的日报经验卡片
3. 逐条对比：新增/去重合并/升级/降级淘汰
4. 更新手册内容 + 更新概览统计

**经验生命周期**：

| 操作 | 条件 | 行为 |
|------|------|------|
| 新增 | 全新经验 | 追加到手册对应分类，★★★☆☆ 起步 |
| 合并 | 与已有条目高度相似 | 合并为一条，标注"多次验证"，★ +1 |
| 升级 | 被 3 次以上引用/验证 | ★ 提升至 ★★★★★ |
| 修正 | 后续实践发现不准确 | 更新内容，标注修正日期和原因 |
| 淘汰 | 被实践否定或技术过时 | 标注"已过时"，移至手册末尾归档区 |

**查看经验手册**时：直接读取 `experience-handbook.md` 并向用户展示。

**搜索经验**时：在经验手册中按关键词匹配标题和内容，返回匹配条目。

### 规则 8：经验推荐（被动触发）

AI 在正常回复过程中，**如果当前对话场景高度命中**经验手册中某条经验，在回复末尾附一行提示：

```
💡 相关经验：EXP-{ID} — {标题}
```

**推荐流程**：
1. 在回复前，根据当前对话的场景分类和关键词，调用搜索脚本：`python {SKILL_DIR}/scripts/experience.py {PROJECT_ROOT} search --keyword "{关键词}"`
2. 从搜索结果中筛选可复用度 ≥ ★★★★☆ 且与当前场景高度匹配的条目
3. 最多选 1 条，附在回复末尾

**推荐条件（必须全部满足）**：
1. 经验手册存在且不为空
2. 搜索脚本返回了匹配结果
3. 当前对话的场景分类与经验条目匹配
4. 该经验可复用度 ≥ ★★★★☆

**约束**：
- 每轮最多推荐 1 条，宁缺毋滥
- 不确定是否相关时不推荐
- 推荐不计入 AI 反馈摘要
- **不需要每轮都调用搜索脚本**，仅在用户问题涉及技术选型、bug 排查、工具使用等明显可匹配经验库的场景时调用

### 规则 9：特殊情况处理

| 情况 | 处理方式 |
|------|---------|
| 用户输入很短（如"ok""确认""好"） | 照常记录，用户输入原封不动 |
| 用户输入包含代码块 | 原封不动记录，保留代码块格式 |
| AI 回复很短（如确认性回复） | 摘要可以只写 1 个要点 |
| 同一天跨多个会话 | 追加到同一天的文件，对话编号继续递增 |
| 日志文件已存在但格式异常 | 追加到文件末尾，不尝试修复已有内容 |
| 写入失败（权限/磁盘） | 向用户提示失败原因，不影响正常对话 |
| 连续多次终止/中断 | 每次终止/中断都独立记录，不合并 |
| 终止后未重新提问就关闭会话 | 无法补记，属于不可控丢失 |
| 日报生成时当日无值得沉淀的经验 | 经验卡片章节写"今日对话以执行为主，暂无新经验沉淀" |
| 经验手册不存在 | 首次生成日报时自动创建 |

### 规则 10：记录行为的透明性

- 每轮对话记录完成后，**不需要**向用户提示"已记录"
- 记录行为是**静默执行**的
- 只有在写入失败时才向用户报告
- 补记不完整对话时也静默执行
- 经验推荐除外（推荐是主动展示给用户的有用信息）

## 脚本工具

Skill 提供以下 Python 脚本用于数据处理，AI 可直接调用：

### report.py — 辅助报告生成

```bash
# 生成日报
python {SKILL_DIR}/scripts/report.py {PROJECT_ROOT} --type daily [--date YYYY-MM-DD]

# 生成周报
python {SKILL_DIR}/scripts/report.py {PROJECT_ROOT} --type weekly [--date YYYY-MM-DD]

# 生成月报
python {SKILL_DIR}/scripts/report.py {PROJECT_ROOT} --type monthly [--date YYYY-MM-DD]
```

解析日志 Markdown 文件，提取结构化数据（对话数、场景分布、时间戳等），输出 JSON 供 AI 填充报告模板。

### experience.py — 经验手册管理

```bash
# 搜索经验
python {SKILL_DIR}/scripts/experience.py {PROJECT_ROOT} search --keyword "{关键词}"

# 统计经验手册概览
python {SKILL_DIR}/scripts/experience.py {PROJECT_ROOT} stats

# 检查去重
python {SKILL_DIR}/scripts/experience.py {PROJECT_ROOT} dedup
```

### stats.py — 跨天统计分析

```bash
# 指定日期范围的统计
python {SKILL_DIR}/scripts/stats.py {PROJECT_ROOT} [--from YYYY-MM-DD --to YYYY-MM-DD]
```

输出：场景分布趋势、对话量趋势、经验增长曲线、完成率统计。

### export.py — 多格式导出（规划中）

> 此脚本尚未实现，作为未来扩展规划。当前如需导出，可直接读取日志/报告的 Markdown 文件。

```bash
# 导出为 CSV/JSON/HTML（规划）
python {SKILL_DIR}/scripts/export.py {PROJECT_ROOT} --format csv|json|html [--date YYYY-MM-DD]
```
