# Spec: conversation-recorder

> **状态**: draft
> **创建日期**: 2026-04-21
> **关联 proposal**: `changes/conversation-recorder-skill/proposal.md`

## 1. 概述

一个 CodeBuddy Project Skill，自动记录用户与大模型的每轮对话内容。用户的自然语言输入原封不动记录，AI 的反馈以关键要点列表形式总结记录。日志按天组织，支持手动生成结构化日报。

**目标用户**：需要定期向领导汇报 AI 协作情况的开发者。

## 2. 功能需求

### F1: 自动对话记录

| 属性 | 描述 |
|------|------|
| **触发条件** | Skill 加载后，AI 在每轮对话结束时自动执行记录 |
| **用户输入记录** | 原封不动记录用户的自然语言文本，保留换行和格式 |
| **AI 反馈记录** | 提取 2-5 个关键要点，每个要点一行，格式为 `- 要点：具体内容` |
| **要点提取规则** | 优先提取：(1)修改/创建了哪些文件 (2)做了什么关键决策 (3)给出了什么建议 (4)执行了什么命令 |
| **存储路径** | `{project_root}/conversation-logs/YYYY-MM-DD.md` |
| **追加模式** | 同一天多轮对话追加到同一文件末尾，用 `---` 分隔 |
| **自动创建目录** | 若 `conversation-logs/` 目录不存在，自动创建 |
| **对话编号** | 当日从 #1 开始递增，通过读取已有文件中最大编号来确定 |
| **时间戳** | 使用当前时间，精确到分钟，格式 `HH:MM` |

### F2: 日报生成

| 属性 | 描述 |
|------|------|
| **触发方式** | 用户手动输入触发词 |
| **触发词** | `生成日报`、`今日总结`、`daily report`、`对话日报`、`生成今日日报` |
| **数据来源** | 读取 `conversation-logs/YYYY-MM-DD.md`（当天日期） |
| **输出路径** | `conversation-logs/reports/YYYY-MM-DD-report.md` |
| **日报结构** | 概览 → 对话摘要表 → 关键产出 → 经验与改进建议 |
| **概览** | 日期、对话总数、涉及项目名称 |
| **对话摘要表** | Markdown 表格：序号、时间、用户问题概括（一句话）、AI 执行摘要（一句话） |
| **关键产出** | 汇总当日所有创建/修改的文件、关键决策 |
| **经验与改进建议** | AI 基于当日对话内容分析总结，提炼可复用经验和待改进点 |
| **自动创建目录** | 若 `reports/` 子目录不存在，自动创建 |

## 3. 页面交互

不适用（本 Skill 无 UI 界面，通过对话交互触发）。

## 4. 质量需求

| 质量属性 | 要求 |
|----------|------|
| **幂等性** | 同一轮对话不会重复记录 |
| **容错性** | 目录不存在时自动创建，写入失败不影响正常对话 |
| **不侵入性** | 日志目录独立于项目源码，不影响 git 工作流（可加入 .gitignore） |
| **无外部依赖** | 纯文件系统操作，不依赖数据库、网络或第三方库 |
| **编码兼容** | 文件统一使用 UTF-8 编码 |

## 5. 功能测试链路

### 5.1 基本流：自动记录对话

```
前置条件：Skill 已加载，conversation-logs/ 目录可能不存在

步骤 1：用户发送一条消息给 AI
  → AI 处理请求并回复用户
  → AI 执行记录逻辑

步骤 2：AI 检查 conversation-logs/ 目录
  → 若不存在，创建目录
  → 确定当日日期文件 YYYY-MM-DD.md

步骤 3：AI 读取当日文件（若存在）
  → 确定当前对话编号（已有最大编号 +1，无则 #1）

步骤 4：AI 构造记录内容
  → 时间戳 [HH:MM]
  → 用户输入（原封不动引用块）
  → AI 反馈摘要（2-5 个关键要点列表）

步骤 5：AI 追加内容到日志文件
  → 验证点：文件末尾新增了一条完整记录
  → 验证点：用户输入与实际发送文本完全一致
  → 验证点：AI 摘要为列表格式，2-5 项
```

### 5.2 基本流：生成日报

```
前置条件：当日已有至少 1 条对话记录

步骤 1：用户说"生成日报"
  → AI 识别触发词

步骤 2：AI 读取 conversation-logs/YYYY-MM-DD.md
  → 解析所有对话记录

步骤 3：AI 生成日报内容
  → 概览：日期、对话总数、项目名
  → 对话摘要表：逐轮概括
  → 关键产出：汇总文件变动和决策
  → 经验建议：分析总结

步骤 4：AI 写入 reports/YYYY-MM-DD-report.md
  → 验证点：日报文件已创建
  → 验证点：包含四个章节
  → 验证点：对话摘要表行数 = 当日对话轮数

步骤 5：AI 向用户展示日报内容
```

### 5.3 备选流：当日无记录时生成日报

```
分支点：基本流 5.2 步骤 2
分支条件：当日日志文件不存在或为空

→ AI 提示用户"今日暂无对话记录，无法生成日报"
→ 不创建空日报文件
```

### 5.4 异常流：文件写入失败

```
触发条件：磁盘空间不足或权限不够
→ AI 提示写入失败原因
→ 不影响当轮对话的正常回复
→ 恢复策略：下一轮对话时重试
```
