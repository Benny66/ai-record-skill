# 测试用例 - conversation-recorder

> **关联 spec**: `specs/conversation-recorder/spec.md`
> **更新日期**: 2026-04-21

## 需求测试用例（TC-FR）

### TC-FR-01: 首次对话自动创建目录和日志文件
- **设计方法**: 等价类划分（首次 vs 非首次）
- **优先级**: P0
- **前置条件**: `conversation-logs/` 目录不存在
- **操作步骤**: 用户发送一条消息并收到 AI 回复
- **预期结果**:
  1. `conversation-logs/` 目录被自动创建
  2. `conversation-logs/YYYY-MM-DD.md` 文件被创建
  3. 文件第一行为 `# 对话记录 - YYYY-MM-DD`
  4. 包含一条 `## [HH:MM] 对话 #1` 记录

### TC-FR-02: 用户输入原封不动记录
- **设计方法**: 等价类划分
- **优先级**: P0
- **前置条件**: Skill 已加载
- **操作步骤**: 用户发送"帮我写一个函数，要求输入是数组，输出是排序后的数组"
- **预期结果**: 日志文件中 `### 用户输入` 下方的 blockquote 内容与用户输入完全一致，一字不差

### TC-FR-03: AI 反馈以关键要点列表记录
- **设计方法**: 等价类划分
- **优先级**: P0
- **前置条件**: Skill 已加载
- **操作步骤**: 用户发送一条需要 AI 执行文件操作的请求
- **预期结果**: `### AI 反馈摘要` 下方有 2-5 个以 `- ` 开头的要点，每个要点包含分类和具体内容

### TC-FR-04: 同一天多轮对话追加记录
- **设计方法**: 边界值分析（第2轮、第N轮）
- **优先级**: P0
- **前置条件**: 当日已有 1 条对话记录
- **操作步骤**: 用户再发送一条消息
- **预期结果**:
  1. 新记录追加到文件末尾
  2. 对话编号为 #2
  3. 第一条记录保持不变

### TC-FR-05: 对话编号正确递增
- **设计方法**: 边界值分析
- **优先级**: P1
- **前置条件**: 当日已有 3 条对话记录（#1, #2, #3）
- **操作步骤**: 用户再发送一条消息
- **预期结果**: 新记录的对话编号为 #4

### TC-FR-06: 时间戳格式正确
- **设计方法**: 等价类划分
- **优先级**: P1
- **前置条件**: Skill 已加载
- **操作步骤**: 用户发送消息
- **预期结果**: 记录中时间戳格式为 `[HH:MM]`，24 小时制

## 日报生成测试用例（TC-DR）

### TC-DR-01: 正常生成日报
- **设计方法**: 场景法（基本流）
- **优先级**: P0
- **前置条件**: 当日已有 3 条对话记录
- **操作步骤**: 用户输入"生成日报"
- **预期结果**:
  1. `conversation-logs/reports/YYYY-MM-DD-report.md` 被创建
  2. 日报包含四个章节：概览、对话摘要、关键产出、经验与改进建议
  3. 概览中对话总数为 3
  4. 对话摘要表有 3 行数据
  5. AI 同时向用户展示日报内容

### TC-DR-02: 各触发词均可触发日报
- **设计方法**: 等价类划分
- **优先级**: P0
- **操作步骤**: 分别输入以下触发词
- **测试数据**:
  | 触发词 | 预期 |
  |--------|------|
  | 生成日报 | 触发日报生成 |
  | 今日总结 | 触发日报生成 |
  | daily report | 触发日报生成 |
  | 对话日报 | 触发日报生成 |
  | 生成今日日报 | 触发日报生成 |

### TC-DR-03: 无记录时生成日报
- **设计方法**: 场景法（备选流）
- **优先级**: P0
- **前置条件**: 当日无对话记录（日志文件不存在）
- **操作步骤**: 用户输入"生成日报"
- **预期结果**:
  1. AI 提示"今日暂无对话记录，无法生成日报"
  2. 不创建日报文件

### TC-DR-04: reports 目录自动创建
- **设计方法**: 边界值分析（首次生成日报）
- **优先级**: P1
- **前置条件**: `conversation-logs/reports/` 目录不存在
- **操作步骤**: 用户输入"生成日报"
- **预期结果**: `reports/` 目录被自动创建，日报文件正常写入

## 错误推测测试用例（TC-EG）

### TC-EG-01: 日志文件格式异常时追加
- **设计方法**: 错误推测
- **优先级**: P2
- **前置条件**: 日志文件存在但内容格式不规范（如被手动编辑过）
- **操作步骤**: 用户发送消息
- **预期结果**: 新记录追加到文件末尾，不尝试修复已有内容

### TC-EG-02: 写入失败不影响对话
- **设计方法**: 错误推测
- **优先级**: P1
- **前置条件**: 日志目录权限为只读
- **操作步骤**: 用户发送消息
- **预期结果**: AI 正常回复用户，提示日志写入失败
